# Code of Conduct

## Our Standard

This project is committed to a respectful, constructive environment for users,
contributors, and maintainers.

Examples of behavior that contribute to a positive environment include:

- Being respectful in technical disagreement
- Giving actionable, evidence-based feedback
- Assuming good intent while still being precise about problems
- Focusing criticism on code, design, and behavior rather than people

Examples of unacceptable behavior include:

- Harassment, insults, or personal attacks
- Discriminatory or exclusionary language
- Doxxing or sharing private information without permission
- Posting credentials, tokens, or private training data in public threads
- Repeatedly derailing technical discussion

## Scope

This code of conduct applies to:

- GitHub issues
- Pull requests
- Code review discussions
- Project documentation and support discussions

## Enforcement

Project maintainers may edit, remove, or reject comments, commits, issues, and
pull requests that do not follow this policy.

## Reporting

For routine conduct problems, open a GitHub issue.

For sensitive situations, do not post private details publicly. Open a minimal
issue requesting a private contact path if one is not already available.
